# GitHub MathJax 公式渲染规则（25项边界测试总结）

> 测试日期：2026-06-13  
> 测试文件：`notes/mathjax-boundary-test.md`  
> 目的：找出 GitHub Markdown 中 `$...$` 和 `$$...$$` 公式渲染失败的确切规律

---

## 一、核心铁律（唯一红线）

**`$` 符号与任何中文标点之间必须有空格隔开。**

| 写法 | 结果 | 正确写法 |
|------|------|----------|
| `：$x$` | ❌ 不渲染 | `： $x$` |
| `（$x$）` | ❌ 不渲染 | `（ $x$ ）` |
| `$x$。）` | ❌ 不渲染 | `$x$ 。）` |
| `$x$，` | ❌ 不渲染 | `$x$ ，` |
| `$x$。` | ❌ 不渲染 | `$x$ 。` |

## 三、安全写法清单

### ✅ 这些写法全部安全（已验证）

1. **多 `$` 同行**：`$A = B$, $C_D$, $\Gamma$, $Z_0$` — 多少对都行
2. **`$` 在行首**：`$x = y$ 是一个等式。` — 没问题
3. **`$` 在行尾**（但后面不是中文标点）：`$V_{in}$` — 行末正常
4. **中文在 `$` 里面**：`$R 和 G 远小于 \omega L$` — 合并写法正常
5. **任意 LaTeX 命令**：`\Gamma`, `\times`, `\parallel`, `\uparrow`, `\downarrow`, `\infty`, `\ln`, `\left`, `\right` — 全部正常
6. **`\text{}` 含中文 in `$$`**：Day 2 已验证
7. **长公式**：任意长度 inline 公式正常
8. **`$` 贴 Markdown 格式**：`**$x$**` 正常

### ❌ 唯一红线：`$` 贴中文标点

```
❌ ：$x$      →  ✅ ： $x$
❌ （$x$）    →  ✅ （ $x$ ）
❌ $x$。      →  ✅ $x$ 。
❌ $x$，      →  ✅ $x$ ，
```

## 四、每次 push 前自检脚本

```bash
python3 -c "
with open('notes/dayXX.md') as f:
    for i,l in enumerate(f,1):
        s=l.rstrip('\n')
        # 检查 $ 贴中文标点
        for p in ['：\$','（\$','\$）','\$，','\$。']:
            if p in s: print(f'L{i}: \$ touching CN punct: {p}')
        # 检查多 $ 同行
        if not s.lstrip().startswith('|') and '\$\$' not in s and s.count('\$')>2:
            print(f'L{i}: MULTI-\$')
"
```

## 五、实战记忆口诀

> **`$` 前后贴中文标点，GitHub 必崩。  
> 加个空格，万事大吉。**

就这么简单。其他所有"看起来可能有问题"的写法——多 `$`、行首 `$`、中文在 `$` 内——其实都没事。
