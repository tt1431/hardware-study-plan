## 1.1 数字信号 vs 模拟信号

先说一个很多人忽视的基本事实：**数字信号是用模拟波形承载的。**

在低速时代（kHz 到几十 MHz），这事不重要。信号边沿缓、带宽窄，PCB 走线近似理想导线，0 就是 0，1 就是 1，工程师只要确保"通不通"即可。

但频率上去以后，电压/电流波形在传输线中的行为完全遵从模拟电路的物理规律——反射、衰减、耦合、谐振，一个不落。0 可能被误判成 1，1 可能被误判成 0。

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-low-vs-high-speed.png)

> 于博士的核心观点：**数字信号活在模拟世界里。** 频率一高，所有模拟效应全部暴露。

### 实践中常见问题

**"面包板上调通了，打板回来就不行"**

面包板和 PCB 完全是两个世界。面包板上寄生电容几十 pF、走线电感不可控，信号边沿被自然"磨圆"了，反而不会振铃。PCB 上走线干净、阻抗可控，边沿陡峭，反射和串扰全部暴露。

**"数字地和模拟地分开，结果更差了"**

很多人学了个"数模地分割"就到处用。分割后如果数字信号跨过分割槽，返回电流无路可走，只能绕大圈——辐射 EMI 暴增。正确的做法是统一参考平面，靠布局分区，而不是切地。

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-ground-partition.png)

**调试技巧：** 用示波器看方波，如果看到振铃、过冲、台阶，那就是模拟效应在作祟——你看的是一个"模拟信号"，虽然它代表的是数字 0 和 1。

## 1.2 一句话概括信号完整性

> 信号完整性（Signal Integrity, SI）：由互连线、电源分配网络、器件等引起的所有信号质量偏差及延时问题。核心就是保证 0/1 码流被准确传到接收端并被准确判断。

于博士把 SI 拆成四大块：

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-si-four-domains.png)

### 实践中常见问题

**"怀疑芯片坏了，换了三颗还是一样"**

十有八九不是芯片的问题。信号质量差（振铃过大）导致接收端误判，换了芯片波形还是那样。先用示波器看信号波形，再怀疑芯片。量过波形再下结论，这是硬件工程师的基本修养。

**"同样的电路，上一版能用，这一版不行"**

很可能是 PCB 布局变了——走线长了 5mm、参考平面被割了一道槽、旁边多了一排高速信号线——任何一个微小的变化都可能引入 SI 问题。SI 从来不是原理图的事，是 Layout 的事。

## 1.3 SI 问题的根源：上升时间

这是第一章最重要的结论，值得反复吃透。

### 表面现象 vs 深层原因

很多人以为"工作频率高了 SI 问题就来了"，这没错但不准确。真正的原因藏在更底层：

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-tr-chain.png)

### 同一个频率，不同的 Tr

同一个 10MHz，Tr 不同，SI 表现天差地别。

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-same-freq-different-tr.png)

SI 设计师看的是**上升时间和带宽**，不是工作频率。

### 实践中常见问题

**"才 25MHz 的时钟，应该没问题吧？"**

大错特错。25MHz 只是工作频率，如果用的是 74LVC 系列驱动器，Tr 可能只有 0.5ns。按 $BW = 0.5/T_r$ 算，有效带宽高达 1GHz。一根 10cm 的走线在 1GHz 下已经不是导线了，是一条传输线。该端接端接，该控阻抗控阻抗。

**"换个慢一点的驱动芯片，振铃就没了"**

慢驱动 = 大 Tr = 小带宽 = 高频分量少 = 反射不严重。这恰恰证明了"Tr 减小是 SI 根源"。但高速接口的 Tr 是规格书定死的——你换不了。

**调试技巧：** 拿到一块新板子，先量时钟信号的上升时间。如果 Tr < 1ns，脑子里 SI 那根弦就得绷紧。

## 1.4 知识体系是网，不是链

于博士在 1.4 小节有段很关键的提醒：单独看每一章都能看懂，但综合运用是另一回事。你为优化一个问题做的操作，可能在其他地方埋了雷。

SI 设计的三个目标天然矛盾：

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-si-triangle.png)

好的 SI 工程师是**平衡艺术家**，不是按图施工的流水工。

### 实践中常见问题

**"串了 33Ω 电阻，振铃确实小了，但数据读不到了"**

串联端接让信号以半幅度传输，末端反射回来才到满幅。如果走线很长，往返延迟太大，接收端长时间处于中间电平——DDR 控制器在这个窗口期采样，读到的就是垃圾。端接解决了反射，但引入了时序问题。

**"加了个磁珠滤电源噪声，纹波更大了"**

磁珠和去耦电容形成 LC 谐振回路。磁珠 SRF 恰好落在噪声频率附近时，不滤波反而放大噪声。换磁珠、加阻尼电阻、或直接换 0Ω——看实际情况。

## 1.5 电平有效 vs 边沿有效

| | 电平有效 | 边沿有效 |
|:---|:---|:---|
| 触发方式 | 高/低电平期间 | 上升/下降沿瞬间 |
| 时序敏感度 | 低 | **高** |
| 典型应用 | EN、CS、RST | SPI 时钟、DDR 时钟 |

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-level-vs-edge.png)

边沿有效对时序敏感的原因：数据只在跳变瞬间被采。时钟上升沿因串扰或反射抖动 50ps，接收端就可能采错。50ps 在低速时代不算什么，在 DDR5 时代就是致命问题。

### 实践中常见问题

**"复位信号加了上拉，上电还是偶尔起不来"**

复位是电平有效信号，对边沿不敏感，但对**电平稳定**敏感。如果上电斜率太慢（RC 充电太长），或电源纹波导致复位引脚在阈值附近反复横跳，芯片可能多次复位。检查上电时序和电源单调性。

**"SPI 跑 10MHz 没问题，50MHz 就丢包"**

不是数据率的问题，是时钟质量问题。50MHz 下时钟边沿更陡，反射和串扰全暴露了。先把时钟信号拉出来看波形。

**调试技巧：** 用示波器余辉模式看时钟，积累几百个周期，如果波形边界模糊、有明显抖动，说明边沿质量不好。

## 1.6 本章涉及的 SI 设计规则

### 高速信号源端串联电阻

很多人背"高速信号串 33Ω"，但不知道为什么。

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-source-termination.png)

原理是源端阻抗匹配。驱动端输出阻抗通常 10~20Ω，加上串联电阻后满足：

$$R_{out} + R = Z_0$$

对于 50Ω 传输线：$17\Omega + 33\Omega = 50\Omega$。信号传输过程：以半幅度传到接收端 → 接收端高阻全反射 → 反射波回来叠加到满幅度 → 回到源端时已匹配，不再反射。

**反射系数公式：**

$$\Gamma = \frac{Z_2 - Z_1}{Z_2 + Z_1}$$

当 $Z_1 = Z_2$ 时 $\Gamma = 0$，无反射。当源端匹配后，反射波回到源端时 $\Gamma = 0$，就此终止。

### 实践中常见问题

**"按照 33Ω 串了，波形还是不好"**

检查三个东西：传输线是不是 50Ω——不是的话用 $R = Z_0 - R_{out}$ 重新算；驱动器输出阻抗是不是你假设的值（不同芯片、电压、温度都在变）；电阻是不是放在离驱动端最近的位置（放远了等于白串）。

**"DDR 线上也串 33Ω？"**

DDR 地址/命令线特性阻抗通常 40Ω 左右，不是 50Ω。33Ω 是错的。查 JEDEC 规范。

**"两个芯片一对一通信，不用串电阻也没事"**

看走线长度。经验法则：走线延时 $> T_r / 6$ 就该端接。$T_r = 0.5\text{ns}$、FR4 信号速度约 $15\text{cm/ns}$ → 临界长度 $\approx 1.25\text{cm}$。超过不端接，等着看振铃。

### 3W 原则

相邻信号线中心间距 ≥ 3 倍线宽（空气间隙 = 2W）。

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-3w-spacing.png)

拉开间距是减小串扰最直接有效的手段。3W 是性价比最优拐点——串扰降到 5% 以下，继续拉大改善有限。

### 实践中常见问题

**"BGA 下面根本做不到 3W"**

BGA 扇出区域是串扰重灾区。退而求其次：逃逸走线尽量短、一出来就散开；关键信号先出两侧用地线隔离；相邻层走线正交；做好端接减小反射噪声对邻线影响。

**"整块板都按 3W 画，结果走不通"**

3W 是建议，不是法律。低速控制信号（GPIO、LED、I2C）不需要 3W；高速时钟和 DDR 数据线才严格遵守。

### 20H 原则

电源平面边缘比地平面边缘**内缩 20 倍层间距**。

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-20h-principle.png)

电源平面和地平面之间构成"平板波导"，电磁波在两层之间来回反射，到板边向外辐射 EMI。电源层内缩后，边缘场被地平面包住——内缩 20H 边缘辐射下降约 70%，继续加大改善不超过 10%。

H 是电源层与地层之间的介质厚度（四层板 L2/L3 之间的 PP 片，通常 0.1~0.2mm）。

**实际工程态度：** 有 DDR/PCIe/射频模块 → 老实做 20H。普通低速板（<100MHz 主频）→ EMC 能过就行。四层板面积紧张 → 优先保证电源完整性，20H 次要。

### 时钟走内层

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-microstrip-vs-stripline.png)

内层走线的两个优势：消除远端串扰（带状线上下介质均匀，奇模偶模速度相等）；屏蔽 EMI（电磁场被上下参考平面约束）。

### 实践中常见问题

**"两层板没有内层，时钟只能走表层"**

两层板时钟走线策略：走线尽可能短；两侧包地铜皮、多打过孔；底层正下方保证完整 GND 平面；远离板边和开关电源。代价是占面积，但两层板本身就做不了太高速。

**"四层板换了八层板，时钟必须换到内层"**

四层板 S-G-P-S 叠层，顶层走时钟 = 表层微带线。速率上去后表层时钟的 EMI 和 FEXT 成为瓶颈。换到内层彻底解决。层数不是拍脑袋定的，是 SI 仿真算出来的。

## 本章小结

SI 核心矛盾：数字信号用模拟波形传输。Tr 缩短是万恶之源。四个子领域互相关联，优化一个可能破坏另一个。设计规则（源端匹配、3W、20H、时钟走内层）要知其所以然。

特性阻抗是 SI 最基础的公式之一：

$$Z_0 = \sqrt{\frac{L}{C}}$$

信号带宽与上升时间的关系：

$$BW = \frac{k}{T_r} \quad (k = 0.35\text{ 或 }0.5)$$

实际工程中，最重要的不是背规则，而是：用示波器一眼看出问题类型；知道何时能破规则、何时绝不能；出了问题能顺着信号链路排查。

## 1.7 SPI 各信号线的阻抗匹配实践

> 📅 2026-06-12 补充·跟天哥闲聊串出来的内容

### 1.7.1 MOSI vs MISO：为什么端接策略不一样

**MOSI（Master → Slave）**
- 单一驱动源（Master），多个接收端
- 未选中的 Slave 处于高阻输入态，不影响信号路径
- 即使有反射，只要在下一个时钟沿前稳定即可，时序裕量大
- ✅ 串阻放在**Master 端**（驱动端），做源端串联匹配

**MISO（Slave → Master）**
- 多个潜在驱动源（每个 Slave 一个输出），虽然同一时间只使能一个
- 激活的 Slave 在 PCB 上的**位置不固定**——选 CS0 和选 CS1，信号源位置不一样
- 未选中的 Slave 输出三态，相当于在主线上挂了**开路 stub**，每个 stub 都是阻抗不连续点
- 切换 Slave 时驱动源变化，反射情况也跟着变
- Master 要在时钟沿采样，反射必须在时钟边沿前 settle 掉，时序更紧
- ✅ 串阻放在**从机端**（驱动端），做源端串联匹配

> **低速/单 Slave 场景：** 两条线是对称的，要匹配一起匹配，不匹配一起拉倒，不必纠结。

### 1.7.2 一个实际案例的配置拆解

| 信号 | 串阻位置 | 串阻值 | 评价 |
|------|---------|--------|------|
| MOSI | MCU 端 | 33Ω | ✅ 驱动端源匹配，标准 |
| MISO | 从机端 | 33Ω | ✅ 驱动端源匹配，标准 |
| CLK | MCU 端 | 33Ω | ✅ 时钟线对 SI 最敏感，必须做 |
| CS | 从机端 | 33Ω | ✅ 限流/保护/抗干扰，逻辑稍有不同 |

**所有串阻都在各自的驱动端**，这是非常规范的处理。33Ω 是 E24 系列标准值，跟大多数器件的输出阻抗加在一起 ≈ 50Ω。

> CS 的 33Ω 放在从机端，跟 MOSI/MISO/CLK 的

源端串联匹配逻辑不完全一样——CS 是电平信号（不是边沿敏感），串阻更多的是做 **RC 滤波 + 限流保护 + 抗毛刺**，不是为了阻抗匹配。

## 1.8 源端串联匹配（Source Series Termination）深入原理

> 33Ω 不是用来消除反射的，而是让驱动端输出阻抗 ≈ 走线特性阻抗，反射波回到驱动端就被吸收，**一个来回结束战斗**。

### 工作原理

假设：
- MCU 推挽输出内阻 $R_{drv} \approx 17\Omega$
- 串阻 $R_s = 33\Omega$
- PCB 走线 $Z_0 = 50\Omega$
- 接收端输入阻抗 $\approx$ 高阻（MΩ 级别）

**过程分解：**

1. **信号发射：** 驱动端总阻抗 $17 + 33 = 50\Omega = Z_0$，信号以半电压 $V_0 \times \frac{50}{50+50} = 0.5V_0$ 进入走线
2. **到达末端：** 遇到高阻负载，反射系数 $\Gamma_L = \frac{\infty - 50}{\infty + 50} \approx +1$，**几乎全反射**
3. **末端叠加：** 入射波 $0.5V_0$ + 反射波 $0.5V_0 = 1.0V_0$，接收端看到完整的满摆幅 ✅
4. **反射返回源端：** 源端阻抗 $50\Omega = Z_0$，$\Gamma_S = 0$，**反射波被吸收，不再反弹** ✅

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-source-termination.png)

> **不加 33Ω 的情况：** 反射波回到源端（$17\Omega \ne 50\Omega$），$\Gamma_S \ne 0$，再次反射→来回弹好几次才衰减完。这就是振铃。

### 时序示意

```
不加 33Ω:  ▁▁╲╱╲╱╲╱╲╱▔▔   来回反射多次才稳
加 33Ω:    ▁▁╲▁▁▁▁▁╱▔▔   一个台阶→反射回来叠满，稳了
```

接收端看到的是：电压先到一半 → 等待走线往返时延 $2T_d$ → 反射回来叠到满幅。

### 注意事项

- 走线太长时往返延迟 $2T_d$ 太大，接收端在窗口期内长时间处于中间电平 → **可能被误采样**
- 源端匹配只适用于**点对点**拓扑（一个驱动端对一个接收端）
- 多负载场景（菊花链/星形）需要不同类型端接

## 1.9 几MHz 下串 33Ω 是否有意义？

### 判断是否需要做传输线匹配的经验法则

**走线长度 > 1/10 信号上升沿的等效物理长度**时，才必须考虑传输线效应。

$$L_{critical} \approx \frac{T_r}{6} \times v_p$$

其中 $v_p$ 是信号在 PCB 上的传播速度（FR4 下约 $15\text{cm/ns}$）

### 举个例子

| 时钟频率 | 上升时间 $T_r$ | 临界长度 | 板子典型走线 | 需要端接吗？ |
|---------|-------------|---------|------------|------------|
| **1 MHz** | ~100 ns | ≈ 250 cm | 5~15 cm | ❌ 不需要 |
| **10 MHz** | ~10 ns | ≈ 25 cm | 5~15 cm | ❌ 通常不需要 |
| **50 MHz** | ~2 ns | ≈ 5 cm | 5~15 cm | ⚠️ 接近临界 |
| **200 MHz** | ~0.5 ns | ≈ 1.25 cm | 5~15 cm | ✅ 必须端接 |

> **关键认知：** 决定要不要做端接的不是**时钟频率**，而是**上升时间/边沿速度**。同一颗 74LVC 芯片跑 1MHz 和 50MHz，T_r 都是纳秒级，边沿一样陡。

### 几 MHz SPI 的场景

- 走线长度远小于临界长度 → **不做端接也没问题**
- 画板子的人串 33Ω 更多是 **设计规范 + 工程习惯 + 心理安慰**
  - 设计规范要求"所有高速输出串 33Ω"，照做
  - 批量备料方便，不管用不用都贴
  - 留着也不坏事，顶多边沿缓一丢丢
  - 万一将来换高速器件，已经预埋了

**实际效果：** 几 MHz 下把 33Ω 短路掉，示波器上看不出波形差异。

## 1.10 特性阻抗 $Z_0$ 深入理解

### 特性阻抗不是电阻

很多人被"阻抗"两个字骗了。**特性阻抗不是直流电阻，用万用表量不出来。**

- **直流电阻** $\propto$ 长度（越长越大）
- **特性阻抗** $Z_0 = \sqrt{L/C}$，由**截面几何**决定，**与长度无关**

### 类比：水管

| | 直流电阻 | 特性阻抗 |
|--|---------|---------|
| 比喻 | 水管内壁的**摩擦力** | 水管的**管径粗细** |
| 长度影响 | 管子越长，摩擦越大 | 管径跟多长没关系 |
| 单位长度 | 每米 x Ω | 每段 dx 的 L/C 比恒定 |

1cm 粗管和 100m 粗管，管径都是同一个数。同理，1cm 50Ω 走线和 1m 50Ω 走线，特性阻抗都是 50Ω。

### 物理本质：信号每往前走一小段 dx 看到的

```
  ──┬──┬──┬──┬──┬──┬──   走线
    C  L  C  L  C  L  C   寄生的单位长度电容 C、电感 L
  ━━━━━━━━━━━━━━━━━━━━   地平面
```

信号每往前走一步，都要给这一段的寄生电容充电、给寄生电感储能。**这种"推起来有多费劲"的感觉就是特性阻抗。**

$$Z_0 = \sqrt{\frac{L}{C}}$$

- L 和 C 都是**每单位长度**的值，除一下长度抵消了 → 与长度无关
- 高频下感抗和容抗远大于导体电阻 R，R 可忽略

### 由什么决定？

```
    线宽 W
  ┌──────┐
  │ 铜线  │  介质厚度 H  ← 到参考地平面的距离
  └──────┘
━━━━━━━━━━━━━━ 参考地平面
```

| 参数变化 | $Z_0$ 变化 | 直觉 |
|---------|-----------|------|
| **线宽 W ↑** | ↓ | 线宽了，电容大了，容易推 |
| **介质厚度 H ↑** | ↑ | 离地远了，电容小了，难推 |
| **介电常数 $\varepsilon_r$ ↑** | ↓ | 板材电容率大，电容大，容易推 |

近似公式：$Z_0 \approx \frac{87}{\sqrt{\varepsilon_r+1.41}} \ln\left(\frac{5.98H}{0.8W+T}\right)$

## 1.11 为什么是 50Ω？

> **50Ω 不是物理定律，是 1930 年代贝尔实验室工程师懒得选边站的结果。**

### 历史故事

1930 年代，AT&T 贝尔实验室研究长距离同轴电缆传输，发现两个相互矛盾的优化目标：

| 目标 | 最优阻抗 | 原因 |
|------|---------|------|
| **最低衰减**（信号损耗最小） | **77Ω** | 电磁场分布恰好使导体+介质损耗之和最小 |
| **最大功率容量**（不击穿前提下的最大功率） | **30Ω** | 低阻抗下电流大，但电压击穿先到 |

折中取中间值 → **50Ω** ✨

### 后续沿用

- 军用/航空 RF 标准全部采用 50Ω 同轴电缆
- 测试仪器（示波器、网络分析仪、信号发生器）输入输出都设计成 50Ω
- PCB 设计者做阻抗控制时自然跟着这个标准
- 到了数字高速时代，"单端 50Ω，差分 100Ω"已经是约定俗成

### 其他常见阻抗值

| 阻抗 | 用途 |
|------|------|
| **50Ω** | 通用 RF / 高速数字单端 |
| **75Ω** | 视频 / 有线电视（更低衰减） |
| **93Ω** | 旧式 IBM 主机总线（更低电容） |
| **100Ω 差分** | HDMI / PCIe / LVDS / 以太网 |
| **90Ω 差分** | USB 3.0（延续 USB 传统） |
| **85Ω 差分** | DDR5 / PCIe Gen5 |
| **28Ω** | DDR4 地址/命令 T 线（为功率） |

## 1.12 差分对的阻抗匹配

### 差分阻抗 vs 单端阻抗

```
        线间距 S
    ┌──────┐      ┌──────┐
    │ + 线  │      │ ─ 线  │
    └──────┘      └──────┘
━━━━━━━━━━━━━━━━━━━━━━━━━━ 参考地平面
```

- **单端 50Ω**：一条线对参考地的阻抗
- **差分 100Ω**：两条线**之间**的阻抗（不是每条对地 50Ω 那么简单！）

> **误区：** 差分 100Ω ≠ 两条 50Ω 简单相加。因为两条线之间有耦合，正线出来的电场被负线吸走一部分 → 对地等效电容变小 → 每条线对地阻抗比 50Ω 高，约 **55~60Ω**。

$$Z_{diff} \approx 2 \times Z_{0,single} \times (1 - 0.48 \times e^{-0.96 \times S/H})$$

### 差分阻抗的调节参数

| 参数变化 | $Z_{diff}$ 变化 | 直觉 |
|---------|---------------|------|
| **线宽 ↑** | ↓ | 电容大了 |
| **介质厚度 ↑** | ↑ | 离地远了 |
| **线间距 ↑** | ↑ | 耦合弱了，$Z_{diff} \to 2\times Z_{0,single}$ |
| **线间距 ↓** | ↓ | 耦合强了，$Z_{diff}$ 降低 |

### 为什么是 100Ω？

跟 50Ω 一样是历史惯例：

1. **50Ω × 2 约等于 100Ω** — 单端推挽自然演化来的
2. **RS-485/422 工业标准** 最早用 120Ω（双绞线特性），数字领域简化成 100Ω
3. **各协议选了它**：

| 协议 | 差分阻抗 | 备注 |
|------|---------|------|
| HDMI | 100Ω | HDMI 联盟定 |
| DisplayPort | 100Ω | VESA 定 |
| PCIe Gen1/2 | 100Ω | PCI-SIG 定 |
| LVDS | 100Ω | National Semi 推的 |
| 100BASE-TX | 100Ω | 跟双绞线一致 |
| USB 2.0 | 90Ω | Intel 定的 |
| USB 3.0 | 90Ω | 延续 USB |
| DDR5 | 85Ω | JEDEC 定 |

### 差分对的端接方式

#### ① 跨接端接（最推荐）

```
发送端                    接收端
+ ────── 走线 ────── + ─┐
                        ├── R = Z_diff（最常用 100Ω）
─ ────── 走线 ────── ─ ─┘
```

- 跨接电阻直接匹配差分阻抗
- 一个 0805/0603 电阻搞定，简洁干净
- 共模信号看到的是两个 R/2 到地，但接收器只关心差模，共模自动忽略

#### ② 双电阻对地端接

```
                    R₁=50Ω
+ ────────────── + ─┐╱╲┴── GND
                      ╱╲
─ ────────────── ─ ─┐╱╲┴── GND
                    R₂=50Ω
```

- $R_1 = R_2 = Z_{0,single}$（通常 50Ω）
- 等效差分阻抗 $Z_{diff} = R_1 + R_2 = 100\Omega$
- 适用场景：接收器输入有偏置电压，跨接电阻会干扰偏置
- 缺点：多一个元件，多一条到地耗电路径，功耗略大

#### ③ 源端串联（差分版，不推荐）

```
发送端                    接收端
+ ── Rs=33Ω ── 走线 ── + ─┐
                           ├── 可选 Rt=100Ω 跨接
─ ── Rs=33Ω ── 走线 ── ─ ─┘
```

- 每条线串 Rs，$R_s \approx Z_{0,single} - R_{drv}$
- **不推荐**的原因：两条线 Rs 必须精确配对（差 1Ω 相位就偏了）；差分对通常距离长，源端效果不如接收端好

### 各协议的实际端接方案

| 协议 | 端接方式 | 典型值 |
|------|---------|-------|
| **LVDS** | 接收端跨接 | 100Ω（必须） |
| **HDMI** | 接收端跨接 + 对地偏置 | 100Ω 跨接 |
| **USB 2.0** | 发送端 D+/D- 各串 27~33Ω，接收端芯片内部 45Ω 对地 | 27~33Ω |
| **USB 3.0** | 交流耦合电容 + 接收端差分端接 | 100Ω 差分 + 对地偏置 |
| **以太网** | 变压器中心抽头端接 | 100Ω 差分 |
| **PCIe** | 交流耦合 + 接收端差分端接 | 100Ω（芯片内部） |
| **RS-485** | 总线两端各跨接 | 120Ω |

### 差分对的真正优势

1. **抗共模噪声：**
   $$V_{out} = (V_+ + noise) - (V_- + noise) = V_+ - V_-$$
   噪声抵消 ✅

2. **抗电磁辐射：** 两条线紧耦合，电磁场互相抵消

3. **对地弹不敏感：** 看的是两线之间的压差，不依赖对地参考

## 本章扩展小结

- **SPI 端接：** MOSI/MISO/CLK 串阻在各自的驱动端做源端匹配；CS 串阻做 RC 滤波保护
- **源端匹配原理：** 半电压发射 + 末端全反射 + 源端吸收 = 一个来回搞定
- **低频不需要匹配：** 走线长度 < $T_r/6$ 的传输速率时不用纠结
- **特性阻抗：** 不是电阻，由截面几何决定，与长度无关
- **50Ω 起源：** 贝尔实验室 77Ω（最低衰减）和 30Ω（最大功率）的折中
- **差分 100Ω：** 不是两条 50Ω 的简单相加，因为耦合导致单端对地阻抗 ≈ 55~60Ω
- **差分端接：** 最简方案 = 接收端跨接一个 100Ω 电阻
