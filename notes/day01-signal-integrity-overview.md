## 1.1 数字信号 vs 模拟信号

### 低速时代的"好日子"

在 kHz 到几十 MHz 的年代，数字电路工程师只需要把发送端和接收端用导线连起来。因为信号边沿很缓（上升时间大），信号带宽很小，PCB 走线表现得就像一根理想导线——不需要考虑寄生参数的干扰。

那时候 0 就是 0，1 就是 1，信号是完整的。

### 高速时代的"现原形"

数字信号想表达的信息是"数字的"——只有 0 和 1。但**承载 0 和 1 的电压/电流波形是模拟的**。

频率一高，电压和电流在传输线中的行为完全遵从模拟电路的物理规律——反射、衰减、耦合、谐振——全来了。

> ⚠️ **核心观点：数字信号活在模拟世界里。** 频率一高，模拟电路的物理效应全部暴露，SI 问题应运而生。

```
低速时代：导线连上就行，"通不通"的问题
高速时代：电磁波传过去，"好不好"的问题
```

## 1.2 一句话概括信号完整性

> **信号完整性（Signal Integrity, SI）**：由于互连线、电源分配网络、器件等引起的所有信号质量偏差及延时问题。核心是保证"0""1"码流能被准确传输到接收端，并被准确判断。

### SI 的四大子领域

| 领域 | 研究内容 | 典型问题 |
|:---|:---|:---|
| **反射 SI** | 阻抗不连续导致信号反弹 | 过冲、下冲、振铃、台阶 |
| **串扰 SI** | 相邻信号通道互相干扰 | 噪声耦合、边沿抖动 |
| **电源完整性（PI）** | 电源分配网络的噪声 | 地弹、同步开关噪声 |
| **时序 SI** | 信号到达时间不匹配 | 建立/保持时间违规 |

> 📌 广义的 SI 涵盖了互连、电源、器件引起的所有信号质量及延时问题。

## 1.3 ⭐ SI 问题的根源：上升时间

这是本章**最重要**的结论。

### 表面现象

- 频率提高了 → SI 问题出现了
- 芯片越来越快 → SI 越来越难搞

### 深层原因

```
芯片工艺进步（28nm → 14nm → 7nm）
    ↓
晶体管沟道长度缩短
    ↓
晶体管开关时间缩短
    ↓
信号上升时间 Tr 越来越小  ← 根源在此
    ↓
信号包含的高频分量增多
    ↓
PCB 互连通道的寄生参数效应显著
    ↓
波形畸变 → SI 问题
```

> 🔑 **关键洞察：不是"工作频率高"导致了 SI 问题，是"上升时间缩短"导致了 SI 问题。**

### 一个反直觉的例子

一个 **10MHz** 的极陡方波（Tr = 0.1ns），它的高频分量比一个 **100MHz** 的缓边沿时钟（Tr = 5ns）还要多。

所以 SI 设计师关心的不是信号的工作频率，而是**上升时间和带宽**。

### 恶性循环

```
芯片越来越快 → Tr 越来越小 → SI 越来越难搞
         ↑                              ↓
         └── 但芯片工艺持续进步，Tr 只会更小 ──┘
```

这就是 SI 工程师的饭碗——只要芯片还在变快，SI 就永远有活干。

## 1.4 ⚠️ 知识体系是网，不是链

于博士在书中特别提醒：

> 你看懂了每一章，但不代表你能综合运用。为优化一个问题所做的操作，可能会带来其他负面影响。

### 举个例子

```
想减小串扰 → 拉开两线间距
    ↓
走线变长 → 延时增大
    ↓
时序可能出问题 → 反而导致系统不稳定
```

### SI 设计的三角矛盾

```
          信号质量（阻抗匹配、端接）
              /\
             /  \
            /    \
           /      \
          /________\
    成本              面积
  （少加层、少加电阻）  （走线短、器件小）
```

好的 SI 工程师是**平衡艺术家**，不是按图施工的流水工。

## 1.5 电平有效 vs 边沿有效

| 对比维度 | 电平有效 | 边沿有效 |
|:---|:---|:---|
| **触发方式** | 高电平或低电平期间有效 | 上升沿或下降沿的瞬间 |
| **对时序敏感度** | 低 | **高** |
| **典型应用** | 使能信号（EN）<br>片选信号（CS）<br>复位信号（RST） | SPI 时钟<br>DDR 时钟<br>触发信号 |
| **噪声容忍度** | 只要电平稳定即可 | 边沿抖动直接影响数据正确性 |

### 为什么边沿有效对时序更敏感？

数据只在跳变瞬间被采样。如果时钟上升沿因串扰或反射抖动了 50ps，接收端就可能采到错误的数据。50ps 在低速时代不算什么，在 DDR5 时代就是致命的。

## 🔧 本章涉及的 SI 设计规则

### 规则 1：高速信号源端串联 33Ω

```
驱动端 ──[33Ω]──── 50Ω传输线 ──── 接收端
```

| 为什么 33Ω？ | 驱动端输出阻抗（~17Ω）+ 串联 33Ω ≈ 50Ω = 传输线特性阻抗 |
|:---|:---|
| **原理** | 源端阻抗匹配，消除反射 |
| **工作过程** | ① 信号半幅度传到接收端<br>② 接收端全反射（+1）<br>③ 反射波回来叠加 → 满幅度<br>④ 源端已匹配 → 不再反射 |
| **副作用** | 源端波形出现"台阶"，台阶宽度 = 信号往返时间 |

> 💡 33Ω 不是固定的！根据你的传输线特性阻抗和驱动器输出阻抗来算。

### 规则 2：3W 原则

```
←── W ──→        ←── 3W ──→        ←── W ──→
┌────────┐                      ┌────────┐
│ 信号线1 │     ← 2W 空气间隙 →  │ 信号线2 │
└────────┘                      └────────┘
```

| 间距 | 串扰量级 | 说明 |
|:---|:---|:---|
| 1W | ~30-40% | 严重，不可接受 |
| 2W | ~10-15% | 勉强，不推荐 |
| **3W** | **~5% 以下** | ✅ 性价比最优拐点 |

> 💡 拉开间距是减小串扰**最有效**的手段。3W 之后继续加大改善有限。

### 规则 3：20H 原则

```
┌──────────────────────────────────┐ ← 地平面（大）
│  ┌────────────────────────┐      │
│  │   电源平面（内缩 20H）   │      │
│  └────────────────────────┘      │
└──────────────────────────────────┘
      ↑── 20H ──→          ←── 20H ──↑
```

| 为什么内缩？ | PCB 板边缘的电磁场向外辐射 → EMI 问题 |
|:---|:---|
| **为什么是 20H？** | 实验证明：内缩 20H 后边缘辐射下降约 70%，继续加大改善不超过 10% |
| **H 是什么？** | 电源平面和地平面之间的介质厚度 |

### 规则 4：时钟信号走内层

| 原因一 | **消除远端串扰**（FEXT）——内层带状线的奇模和偶模速度相同，远端串扰近似为 0 |
|:---|:---|
| **原因二** | **屏蔽 EMI 辐射**——内层走线被上下参考平面包住，电磁场不外泄 |

> 📌 四条规则都指向同一个核心：理解背后的电磁场原理，比背规则重要一百倍。
