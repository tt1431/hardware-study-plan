> 📅 2026-06-10

## 1.1 数字信号 vs 模拟信号

先说一个很多人忽视的基本事实：**数字信号是用模拟波形承载的。**

在低速时代（kHz 到几十 MHz），这事不重要。信号边沿缓、带宽窄，PCB 走线近似理想导线，0 就是 0，1 就是 1，工程师只要确保"通不通"即可。

但频率上去以后，电压/电流波形在传输线中的行为完全遵从模拟电路的物理规律——反射、衰减、耦合、谐振，一个不落。0 可能被误判成 1，1 可能被误判成 0。

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-low-vs-high-speed.png)

> 于博士的核心观点：**数字信号活在模拟世界里。** 频率一高，所有模拟效应全部暴露。

### 实践中常见问题

**"面包板上调通了，打板回来就不行"**

面包板和 PCB 完全是两个世界。面包板上寄生电容几十 pF、走线电感不可控，信号边沿被自然"磨圆"了，反而不会振铃。PCB 上走线干净、阻抗可控，边沿陡峭，反射和串扰全部暴露。

**"数字地和模拟地分开，结果更差了"**

很多人学了个"数模地分割"就到处用。分割后如果数字信号跨过分割槽，返回电流无路可走，只能绕大圈——辐射 EMI 暴增。正确的做法是统一参考平面，靠布局分区，而不是切地。

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-ground-partition.png)

**调试技巧：** 用示波器看方波，如果看到振铃、过冲、台阶，那就是模拟效应在作祟——你看的是一个"模拟信号"，虽然它代表的是数字 0 和 1。

## 1.2 一句话概括信号完整性

> 信号完整性（Signal Integrity, SI）：由互连线、电源分配网络、器件等引起的所有信号质量偏差及延时问题。核心就是保证 0/1 码流被准确传到接收端并被准确判断。

于博士把 SI 拆成四大块：

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-si-four-domains.png)

### 实践中常见问题

**"怀疑芯片坏了，换了三颗还是一样"**

十有八九不是芯片的问题。信号质量差（振铃过大）导致接收端误判，换了芯片波形还是那样。先用示波器看信号波形，再怀疑芯片。量过波形再下结论，这是硬件工程师的基本修养。

**"同样的电路，上一版能用，这一版不行"**

很可能是 PCB 布局变了——走线长了 5mm、参考平面被割了一道槽、旁边多了一排高速信号线——任何一个微小的变化都可能引入 SI 问题。SI 从来不是原理图的事，是 Layout 的事。

## 1.3 SI 问题的根源：上升时间

这是第一章最重要的结论，值得反复吃透。

### 表面现象 vs 深层原因

很多人以为"工作频率高了 SI 问题就来了"，这没错但不准确。真正的原因藏在更底层：

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-tr-chain.png)

### 同一个频率，不同的 Tr

同一个 10MHz，Tr 不同，SI 表现天差地别。

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-same-freq-different-tr.png)

SI 设计师看的是**上升时间和带宽**，不是工作频率。

### 实践中常见问题

**"才 25MHz 的时钟，应该没问题吧？"**

大错特错。25MHz 只是工作频率，如果用的是 74LVC 系列驱动器，Tr 可能只有 0.5ns。按 BW=0.5/Tr 算，有效带宽高达 1GHz。一根 10cm 的走线在 1GHz 下已经不是导线了，是一条传输线。该端接端接，该控阻抗控阻抗。

**"换个慢一点的驱动芯片，振铃就没了"**

慢驱动 = 大 Tr = 小带宽 = 高频分量少 = 反射不严重。这恰恰证明了"Tr 减小是 SI 根源"。但高速接口的 Tr 是规格书定死的——你换不了。

**调试技巧：** 拿到一块新板子，先量时钟信号的上升时间。如果 Tr < 1ns，脑子里 SI 那根弦就得绷紧。

## 1.4 知识体系是网，不是链

于博士在 1.4 小节有段很关键的提醒：单独看每一章都能看懂，但综合运用是另一回事。你为优化一个问题做的操作，可能在其他地方埋了雷。

SI 设计的三个目标天然矛盾：

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-si-triangle.png)

好的 SI 工程师是**平衡艺术家**，不是按图施工的流水工。

### 实践中常见问题

**"串了 33Ω 电阻，振铃确实小了，但数据读不到了"**

串联端接让信号以半幅度传输，末端反射回来才到满幅。如果走线很长，往返延迟太大，接收端长时间处于中间电平——DDR 控制器在这个窗口期采样，读到的就是垃圾。端接解决了反射，但引入了时序问题。

**"加了个磁珠滤电源噪声，纹波更大了"**

磁珠和去耦电容形成 LC 谐振回路。磁珠 SRF 恰好落在噪声频率附近时，不放滤波反而放大噪声。换磁珠、加阻尼电阻、或直接换 0Ω——看实际情况。

## 1.5 电平有效 vs 边沿有效

| | 电平有效 | 边沿有效 |
|:---|:---|:---|
| 触发方式 | 高/低电平期间 | 上升/下降沿瞬间 |
| 时序敏感度 | 低 | **高** |
| 典型应用 | EN、CS、RST | SPI 时钟、DDR 时钟 |

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-level-vs-edge.png)

边沿有效对时序敏感的原因：数据只在跳变瞬间被采。时钟上升沿因串扰或反射抖动 50ps，接收端就可能采错。50ps 在低速时代不算什么，在 DDR5 时代就是致命问题。

### 实践中常见问题

**"复位信号加了上拉，上电还是偶尔起不来"**

复位是电平有效信号，对边沿不敏感，但对**电平稳定**敏感。如果上电斜率太慢（RC 充电太长），或电源纹波导致复位引脚在阈值附近反复横跳，芯片可能多次复位。检查上电时序和电源单调性。

**"SPI 跑 10MHz 没问题，50MHz 就丢包"**

不是数据率的问题，是时钟质量问题。50MHz 下时钟边沿更陡，反射和串扰全暴露了。先把时钟信号拉出来看波形。

**调试技巧：** 用示波器余辉模式（persistence mode）看时钟，积累几百个周期，如果波形边界模糊、有明显抖动，说明边沿质量不好。

## 1.6 本章涉及的 SI 设计规则

### 高速信号源端串联 33Ω

很多人背"高速信号串 33Ω"，但不知道为什么。

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-source-termination.png)

原理是源端阻抗匹配：驱动端输出阻抗通常 10~20Ω，加上 33Ω 后接近 50Ω。信号以半幅度传到接收端 → 接收端高阻全反射 → 反射波回来叠加到满幅度 → 回到源端时已匹配不再反射。

### 实践中常见问题

**"按照 33Ω 串了，波形还是不好"**

检查三个东西：传输线是不是 50Ω（不是的话 R = Z₀ - Rout 重新算）；驱动器输出阻抗是不是你假设的值；电阻是不是放在离驱动端最近的位置（放远了等于白串）。

**"DDR 线上也串 33Ω？"**

DDR 地址/命令线特性阻抗通常 40Ω 左右，不是 50Ω。33Ω 是错的。查 JEDEC 规范。

**"两个芯片一对一通信，不用串电阻也没事"**

看走线长度。经验法则：走线延时 > Tr/6 就该端接。Tr=0.5ns、FR4 信号速度约 15cm/ns → 临界长度 ≈ 1.25cm。超过这个长度不端接，等着看振铃。

### 3W 原则

相邻信号线中心间距 ≥ 3 倍线宽（空气间隙 = 2W）。

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-3w-spacing.png)

拉开间距是减小串扰最直接有效的手段。3W 是性价比最优拐点。

### 实践中常见问题

**"BGA 下面根本做不到 3W，怎么办？"**

BGA 扇出区域是串扰重灾区。退而求其次：逃逸走线尽量短、一出来就散开；关键信号先出两侧用地线隔离；相邻层走线正交；做好端接减小反射噪声对邻线影响。

**"整块板都按 3W 画，结果走不通"**

3W 是建议，不是法律。低速控制信号（GPIO、LED、I2C）不需要 3W；高速时钟和 DDR 数据线才严格遵守。

### 20H 原则

电源平面边缘比地平面边缘**内缩 20 倍层间距**。

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-20h-principle.png)

电源平面和地平面之间构成"平板波导"，电磁波在两层之间来回反射，到板边向外辐射 EMI。电源层内缩后，边缘场被地平面包住——内缩 20H 边缘辐射下降约 70%，继续加大改善不超过 10%。20H 是"够用不浪费"的拐点。

H 是电源层与地层之间的介质厚度（四层板 L2/L3 之间的 PP 片，通常 0.1~0.2mm）。

**实际工程态度：** 有 DDR/PCIe/射频模块 → 老实做 20H。普通低速板（<100MHz 主频）→ EMC 能过就行。四层板面积紧张 → 优先保证电源完整性，20H 次要。

### 时钟走内层

![](https://raw.githubusercontent.com/tt1431/hardware-study-plan/main/assets/01-microstrip-vs-stripline.png)

内层走线的两个优势：消除远端串扰（带状线上下介质均匀，奇模偶模速度相等）；屏蔽 EMI（电磁场被上下参考平面约束）。

### 实践中常见问题

**"两层板没有内层，时钟只能走表层"**

两层板时钟走线策略：走线尽可能短；两侧包地铜皮、多打过孔；底层正下方保证完整 GND 平面；远离板边和开关电源。代价是占面积，但两层板本身就做不了太高速。

**"四层板换了八层板，时钟必须换到内层"**

四层板 S-G-P-S 叠层，顶层走时钟 = 表层微带线。速率上去后表层时钟的 EMI 和 FEXT 成为瓶颈。换到内层彻底解决。层数不是拍脑袋定的，是 SI 仿真算出来的。

## 本章小结

SI 核心矛盾：数字信号用模拟波形传输。Tr 缩短是万恶之源。四个子领域互相关联，优化一个可能破坏另一个。设计规则（33Ω/3W/20H/时钟走内层）要知其所以然。

实际工程中，最重要的不是背规则，而是：用示波器一眼看出问题类型；知道何时能破规则、何时绝不能；出了问题能顺着信号链路排查。后面每一章慢慢积累。
