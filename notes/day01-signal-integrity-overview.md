# 信号完整性学习笔记 ①｜什么是信号完整性？

> 📚 《信号完整性揭秘：于博士 SI 设计手记》  
> 📅 2026-06-10

---

## 1.1 数字信号 vs 模拟信号

先说一个很多人忽视的基本事实：**数字信号是用模拟波形承载的。**

在低速时代（kHz 到几十 MHz），这事不重要。信号边沿缓、带宽窄，PCB 走线近似理想导线，0 就是 0，1 就是 1，工程师只要确保"通不通"即可。

但频率上去以后，电压/电流波形在传输线中的行为完全遵从模拟电路的物理规律——反射、衰减、耦合、谐振，一个不落。0 可能被误判成 1，1 可能被误判成 0。

> 于博士的核心观点：**数字信号活在模拟世界里。** 频率一高，所有模拟效应全部暴露。

### 实践中常见问题

**"面包板上调通了，打板回来就不行"**

面包板和 PCB 完全是两个世界。面包板上寄生电容几十 pF、走线电感不可控，信号边沿被自然"磨圆"了，反而不会振铃。PCB 上走线干净、阻抗可控，边沿陡峭，反射和串扰全部暴露。

**"数字地和模拟地分开，结果更差了"**

很多人学了个"数模地分割"就到处用。分割后如果数字信号跨过分割槽，返回电流无路可走，只能绕大圈——辐射 EMI 暴增。正确的做法是统一参考平面，靠布局分区，而不是切地。

**调试技巧：用示波器看方波，如果看到振铃、过冲、台阶，那就是模拟效应在作祟——你在看的是一个"模拟信号"，虽然它代表的是数字 0 和 1。**

## 1.2 一句话概括信号完整性

> 信号完整性（Signal Integrity, SI）：由互连线、电源分配网络、器件等引起的所有信号质量偏差及延时问题。核心就是保证 0/1 码流被准确传到接收端并被准确判断。

于博士把 SI 拆成四大块：

| 领域 | 研究什么 | 典型问题 |
|:---|:---|:---|
| 反射 SI | 阻抗不连续引起信号反弹 | 过冲、下冲、振铃、台阶 |
| 串扰 SI | 相邻通道互相干扰 | 噪声耦合、边沿抖动 |
| 电源完整性 | 电源分配网络噪声 | 地弹、同步开关噪声 |
| 时序 SI | 信号到达时间不一致 | 建立/保持时间违规 |

### 实践中常见问题

**"怀疑芯片坏了，换了三颗还是一样"**

十有八九不是芯片的问题。信号质量差（振铃过大）导致接收端误判，换了芯片波形还是那样。先用示波器看信号波形，再怀疑芯片。量过波形再下结论，这是硬件工程师的基本修养。

**"同样的电路，上一版能用，这一版不行"**

很可能是 PCB 布局变了——走线长了 5mm、参考平面被割了一道槽、旁边多了一排高速信号线——任何一个微小的变化都可能引入 SI 问题。SI 从来不是原理图的事，是 Layout 的事。

## 1.3 SI 问题的根源：上升时间

这是第一章最重要的结论，值得反复吃透。

### 表面现象 vs 深层原因

很多人以为"工作频率高了 SI 问题就来了"，这没错但不准确。真正的原因藏在更底层：

```
芯片工艺进步（28nm → 14nm → 7nm）
    ↓
晶体管沟道长度缩短
    ↓
晶体管开关时间缩短
    ↓
信号上升时间 Tr 越来越小  ← 根源
    ↓
高频分量增多
    ↓
PCB 寄生参数效应暴露
    ↓
波形畸变
```

举个反直觉的例子：一个 **10MHz** 的极陡方波（Tr=0.1ns），高频分量比一个 **100MHz** 缓边沿时钟（Tr=5ns）还多。SI 设计师看的是上升时间和带宽，不是工作频率。

### 实践中常见问题

**"才 25MHz 的时钟，应该没问题吧？"**

大错特错。25MHz 只是工作频率，如果用的是 74LVC 系列驱动器，Tr 可能只有 0.5ns。按 BW=0.5/Tr 算，有效带宽高达 1GHz。一根 10cm 的走线在 1GHz 下已经不是导线了，是一条传输线。该端接端接，该控阻抗控阻抗。

**"换个慢一点的驱动芯片，振铃就没了，但为什么还能正常工作？"**

不是"还能正常工作"——是**正因为换慢了才正常工作**。慢驱动 = 大 Tr = 小带宽 = 高频分量少 = 反射不严重。这恰恰证明了"Tr 减小是 SI 根源"这个论点。但你不是每次都能换慢芯片——高速接口的 Tr 是规格书定死的。

**调试技巧：** 拿到一块新板子，先量时钟信号的上升时间。如果 Tr < 1ns，脑子里 SI 那根弦就得绷紧。

## 1.4 知识体系是网，不是链

于博士在 1.4 小节有段很关键的提醒：单独看每一章都能看懂，但综合运用是另一回事。你为优化一个问题做的操作，可能在其他地方埋了雷。

举个例子：

```
想减小串扰 → 拉开两线间距 → 走线变长 → 延时增大 → 时序出问题
```

SI 设计的三个目标天然矛盾：

- 信号质量（阻抗匹配、端接）
- 成本（少加层、少加器件）
- 面积（走线短、器件小）

好的 SI 工程师是**平衡艺术家**，不是按图施工的流水工。

### 实践中常见问题

**"串了 33Ω 电阻，振铃确实小了，但数据读不到了"**

串联端接让信号以半幅度传输，末端反射回来才到满幅。如果走线很长，往返延迟太大，接收端长时间处于中间电平——DDR 控制器在这个窗口期采样，读到的就是垃圾。端接解决了反射，但引入了时序问题。这就是"拆东墙补西墙"。

**"加了个磁珠滤电源噪声，纹波更大了"**

磁珠和电源平面上的去耦电容形成了一个 LC 谐振回路。如果磁珠的自谐振频率恰好落在板子上的某个噪声频率附近，不仅不滤波，反而把噪声放大了。换磁珠、加阻尼电阻、或者直接去掉磁珠用 0Ω——看情况。

## 1.5 电平有效 vs 边沿有效

| | 电平有效 | 边沿有效 |
|:---|:---|:---|
| 触发方式 | 高/低电平期间 | 上升/下降沿瞬间 |
| 时序敏感度 | 低 | **高** |
| 典型应用 | EN、CS、RST | SPI 时钟、DDR 时钟 |

边沿有效对时序敏感的根本原因：数据只在跳变瞬间被采。时钟上升沿因串扰或反射抖动 50ps，接收端就可能采到错误数据。50ps 在低速时代不算什么，在 DDR5 时代就是致命问题。

### 实践中常见问题

**"复位信号加了上拉，上电还是偶尔起不来"**

复位是电平有效信号，对边沿不敏感，但对**电平稳定**敏感。如果上电斜率太慢（RC 充电时间太长），或者电源纹波导致复位引脚在阈值电压附近反复横跳，芯片可能多次复位。检查上电时序和电源单调性。

**"SPI 跑 10MHz 没问题，50MHz 就丢包"**

不是数据率的问题，是时钟质量问题。50MHz 下时钟边沿更陡，走线上的反射和串扰全暴露了——接收端看到的不是干净的时钟，而是带着振铃和台阶的波形。每一次误触发就丢一个 bit。先把时钟信号拉出来看波形。

**调试技巧：** 用示波器的余辉模式（persistence mode）看时钟信号，积累几百个周期，如果波形边界模糊、有明显抖动，说明边沿质量不好。单次触发很难发现间歇性的抖动。

## 1.6 本章涉及的 SI 设计规则

### 高速信号源端串联 33Ω

很多人背"高速信号串 33Ω"，但不知道为什么。

```
驱动端 ──[33Ω]──── 50Ω传输线 ──── 接收端
```

原理是源端阻抗匹配：驱动端输出阻抗通常 10~20Ω，加上 33Ω 后接近 50Ω，和传输线特性阻抗相等。信号先以半幅度传到接收端 → 接收端高阻全反射 → 反射波回来叠加到满幅度 → 回到源端时已匹配不再反射。

### 实践中常见问题

**"按照 33Ω 串了，波形还是不好"**

检查三个东西：第一，你的传输线是不是 50Ω——如果不是，33Ω 是错的（算一下：R = Z₀ - Rout）。第二，驱动器的输出阻抗是不是你假设的值——不同芯片、不同供电电压、不同温度下 Rout 都在变。第三，串联电阻有没有放在离驱动端最近的位置——放错了位置等于白串。

**"DDR 线上也串 33Ω？"**

DDR 的地址/命令线特性阻抗通常是 40Ω 左右，不是 50Ω。33Ω 是错的。查 JEDEC 规范和你用的 DDR 芯片手册，不要照搬。

**"两个芯片一对一通信，不用串电阻也没事"**

看走线长度。经验法则：走线延时 > Tr/6 就该端接。Tr=0.5ns、FR4 上信号速度约 15cm/ns → 临界长度 = 15 × 0.5/6 ≈ 1.25cm。超过 1.25cm 还没端接，等着看振铃吧。

### 3W 原则

相邻信号线中心间距 ≥ 3 倍线宽。

### 实践中常见问题

**"BGA 下面根本做不到 3W，怎么办？"**

BGA 扇出区域是串扰的重灾区。做不到 3W 就做好以下几件事：① BGA 的逃逸走线尽量短，一出来就散开；② 关键信号（时钟、复位）先出，两侧用地线隔离；③ 相邻层走线正交互不重叠；④ 做好端接，减小反射噪声对相邻线的影响。退而求其次，控制最坏情况。

**"整块板都按 3W 画，结果走不通"**

3W 是建议，不是法律。根据实际信号速率和上升时间判断：低速控制信号（GPIO、LED、I2C）不需要 3W；高速时钟和 DDR 数据线才需要严格遵守。学会用仿真验证你的实际间距是否够用。

### 20H 原则

通常 4 层板很难严格做到 20H（电源层边缘距板边太远，浪费铜皮）。实际工程中电源层和地层边缘齐平的板子多了去了，只要 EMC 测试能过就行。如果板子频率不高（<100MHz 主频），20H 不是硬性要求。但如果板子上有 500MHz+ 的 DDR 时钟或者射频模块，老实用 20H。

### 时钟走内层

### 实践中常见问题

**"两层板没有内层，时钟只能走表层"**

两层板的时钟走线策略：① 走线尽可能短，越短越好；② 包地（两边加 GND 铜皮，多打过孔）；③ 底层正下方保证是完整 GND 平面（不要被其他走线割断）；④ 远离板边、远离连接器和开关电源。代价是占面积，但两层板本身就做不了太高速的东西。

**"四层板换了八层板，工程师说时钟必须换到内层——为什么？"**

四层板的常用叠层是 S-G-P-S（信号-地-电源-信号），顶层走时钟等于表层微带线。如果板子频率不高，这样没问题。但一旦速率上去（DDR3 以上），表层时钟的 EMI 辐射和 FEXT 串扰就会成为瓶颈。换到内层（比如六层板的 Layer 3，上下有参考平面）可以彻底解决。层数不是拍脑袋决定的，是 SI 仿真算出来的。

## 📝 本章小结

SI 核心矛盾：数字信号用模拟波形传输。Tr 缩短是万恶之源。四个子领域（反射/串扰/PI/时序）互相关联，优化一个可能破坏另一个。设计规则（33Ω/3W/20H/时钟走内层）要知其所以然，否则换个场景就不知道该怎么调了。

实际工程中，最重要的能力不是背规则，而是：① 示波器量波形一眼看出是什么问题（反射？串扰？电源噪声？）；② 知道什么情况下可以打破规则、什么情况下绝对不能；③ 出了问题能顺着信号链路一步步排查。这些，后面每一章慢慢积累。
